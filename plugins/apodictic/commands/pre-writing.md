description: Guide a writer from idea to draftable structure (no manuscript required)
allowed-tools: Read, Write, Edit, Bash, Glob
---

Run the Pre-Writing Pathway for a writer who has an idea but no manuscript yet.

Load the `pre-writing-pathway` skill. Then:

1. **Phase 0: Writer Mode Calibration** — Identify whether this writer is architecture-first or discovery-first. Don't ask directly; listen for signals.

2. **Phase 1: Seed Inventory** — Ask the writer to talk about their idea. Map what they say onto the seed schema (premise, character, world, feeling, scene, theme, genre instinct, comp). Report back what you heard and what's still open. Begin the Uncertainty/Assumption Ledger.

3. **Phase 2: Controlling Idea** — Surface the value + cause formulation. If the writer doesn't know yet, mark as Unknown and move on.

4. **Phase 3: Protagonist Engine** — Build the minimum viable character architecture (want, need, lie, fear, want-need tension). Test the **Storyable Gate** before proceeding.

5. **Phase 4: Spine Selection** — Present 2–3 viable structural candidates with tradeoffs (Option Architecture). Don't converge to one recommendation prematurely. Load the `plot-architecture` skill at this phase for spine selection.

6. **Phase 4B: Complexity Budget** — Set explicit scope caps for the first draft.

7. **Phase 5: Structural Plan** — Test the **Draftable Gate**. If it passes, generate the full Pre-Drafting Structural Plan. If it fails for a discovery-first writer, generate a Minimal Viable Plan instead.

8. **Phase 6: Exit Ramp** — Deliver the plan and explain re-entry triggers.

Save outputs to `Outputs/[Project]/` using Core DE naming conventions.

Load the `core-editor` skill only if the writer requests re-entry into a development edit workflow (e.g., they have a draft and want structural diagnosis). Do not preload it at session start.

If a project name is provided as text, use it for the project directory and file naming.
