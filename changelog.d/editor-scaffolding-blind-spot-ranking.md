### Operators — Editor Scaffolding blind-spot ranking

Editor Scaffolding can now **rank the "What You Might Have Missed" section by blind-spot gap** — the distance between a finding's structural severity and its surface *salience* (how easy it is to miss) — rather than by severity alone, so a high-severity finding that a confident read glides past (masked by strong prose) leads, not the obvious ones the editor already caught. A new **optional `salience` field** on `apodictic.finding.v1` (`prominent` / `moderate` / `subtle`) carries the miss-risk judgment, and a new `apodictic.blind_spot_ranking.v1` block — one per entry, in display order, each `{finding_id, severity, salience}` (mirroring how `apodictic.severity_calibration.v1` duplicates `delivered`) — declares the order. An **opt-in** sub-check on the existing `editor-scaffolding` validator extends both the letter's E2 and the per-pass P2: R1 each block schema-valid, R2 descending-gap order (`gap = rank(severity) − rank(salience)`, Must-Fix/prominent = 3 … Could-Fix/subtle = 1; ties break by descending severity), and R3 (with an optional `--ledger=<ledger.md>` companion) each `finding_id` exists in the ledger and its severity matches the locked tier — a laundered severity fails. A scaffolded letter or pass *without* ranking blocks stays valid (unranked), so existing fixtures are not retro-broken. The model owns severity *and* salience; the validator checks only the gap arithmetic and sort order — no prose is read. Ranked blocks are added to both canonical worked examples (`example-editorial-letter-scaffolded.md`, `example-pass-scaffolded.md` under `--strict`) and gated under `--check-all`; the new schema is bound to `editor-scaffolding` in `schemas/_coverage.json`. No new validator — the self-testable validator count is unchanged (67).
