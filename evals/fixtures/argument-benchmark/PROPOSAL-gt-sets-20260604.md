## The principle

`GT3 — Strongest real objection` (and, on Roosevelt, `GT2 — Failure locus`)
registers **one** answer. But a competent argument often has **more than one
genuinely structurally-prior** objection/locus, and two independent runs can each
land on a different valid one. A single registered answer then mislabels a
*defensible alternate read* as an engine miss — conflating "engine wrong" with
"key too narrow."

**Reform:** register the locus / objection as a **set** of acceptable
structurally-prior answers, each with its code and a one-line structural
justification.

### The discipline (this is the whole point — four guards)

1. **IN criterion — structural priority, not salience.** A member qualifies only
   if it is *logically upstream of the conclusion* or *turns the argument's own
   warrant against it*. "A reader would raise this first" is **not** sufficient.
2. **OUT list stays explicit.** Decoys and merely-secondary objections remain
   registered as **misses** (OB5, etc.). **Decoys stay decoys.**
3. **Inclusion is earned by structural argument, not by a run finding it.** Two
   engines agreeing on a candidate is *corroboration* that it's defensible — it is
   **not** a license to admit it. Each member below carries an independent reason
   it belongs, written before checking whether it makes a run pass.
4. **Convergence guard.** After expanding, re-verify that fixtures which *should*
   fail **still fail**. If a set expansion flips a non-converging fixture to
   "pass" *only* by blessing a run's error, **reject the expansion.**
   → **None of the three changes below flips a fixture to converge.** Roosevelt
   and PPI still fail after the edits (see the impact table). That is the evidence
   the discipline held, not a coincidence.
